# Topic: Contextual Guidance for Interactive Tools

## Question
What should "what to notice" prompts look like for each interactive tool? Not just UX copy — what are the pedagogically important observations at each tool that would turn parameter-fiddling into genuine understanding?

## Context
The 2026-02-21 visitor flagged the Coherence Explorer as HIGH severity: "No explanation of what I'm looking at or why it matters. The tool assumes I already know the theory. A one-sentence 'what to notice' prompt would help enormously." They also noted the Scale Navigator needed more visual feedback, and the γ Calculator was more intuitive thanks to its presets.

The pattern across tools: tools that ground abstract math in physical systems (γ Calculator presets, Galaxy Plotter) work well. Tools that present raw mathematical exploration without context (Coherence Explorer) lose the visitor.

## Why It Matters
Interactive tools are the site's strongest differentiator — no other physics framework site has them. But they're only effective if visitors know what to observe. The Scale Navigator was the visitor's "aha moment" — imagine if every tool could produce that kind of insight.

## Suggested Starting Points
- Live site: all 10 tools in the Interactive Tools category
- Focus on these tools that need the most guidance:
  - Coherence Explorer: What should you notice when γ changes? When ρ_crit changes?
  - Scale Navigator: What's the key observation at biological scales?
  - Phase Boundary Visualizer: What does each region mean physically?
  - Equation Walkthrough: What's the key insight at each step?
- Tools that already work well (study why):
  - γ Calculator: presets ground the math in physical systems
  - Galaxy Plotter: the gap between Newtonian and observed is immediately visual
  - Prediction Tracker: filtering by status gives agency
- Design question: should guidance be always-visible, toggleable, or shown only on first visit?
- Consider difficulty tiers: beginner guidance vs. advanced "things to explore"
